## YouTube — Why It Doesn't Work (and How to Fix It Later)

YouTube now requires a **poToken** (Proof of Origin Token) in addition to cookies. This token is generated by a service called `yt-session-generator` which runs a headless Chromium browser to solve Google's BotGuard challenge.

**The problem**: Railway's network blocks the outbound connections needed for BotGuard — the yt-session-generator times out every time it tries to generate a token.

**The fix (for when you have a VPS or home server)**:

1. Run both `cobalt` and `yt-session-generator` on the same server (they must share the same public IP):

```bash
# yt-session-generator (webserver mode)
docker run -d \
  --name yt-session-gen \
  -p 8080:8080 \
  ghcr.io/imputnet/yt-session-generator:webserver

# cobalt
docker run -d \
  --name cobalt \
  -p 9000:9000 \
  -e API_URL="https://your-domain.com/" \
  -e CORS_WILDCARD=1 \
  -e COOKIE_PATH=/cookies.json \
  -e YOUTUBE_SESSION_SERVER="http://localhost:8080" \
  -v /path/to/cookies.json:/cookies.json \
  ghcr.io/imputnet/cobalt:11
```

2. Update the PWA Settings with the new cobalt URL.

**cookies.json location**: `Z:\Elton\My Vibe Code Projects\Black Hole\cobalt-instance\cookies.json`
> ⚠️ YouTube cookies expire periodically. Re-export from browser if YouTube stops working after moving to VPS.
